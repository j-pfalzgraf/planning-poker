# ? Story Points – Esforço Desconhecido

> **Esforço:** Não pode ser estimado
> **Risco:** Desconhecido
> **Ação:** Spike/Pesquisa necessária

---

## Quando "?" é Usado?

A carta "?" sinaliza que uma história **não pode ser estimada** porque:

- Requisitos não estão claros
- Viabilidade técnica é incerta
- Existem muitas incógnitas
- Mais contexto é necessário

---

## Exemplo 1: Requisitos Não Claros

**Título:** Importação de Dados do Sistema Legado

**Descrição:**
Dados do sistema antigo devem ser importados.

**Por que "?":**

- Em que formato estão os dados?
- Quantos registros?
- Quais campos são mapeados?
- Existem regras de validação?

**Próximo Passo:**
Reunião com stakeholder para esclarecer requisitos.

---

## Exemplo 2: Incerteza Técnica

**Título:** Integração com API de Terceiro

**Descrição:**
Conectar à API do fornecedor XY.

**Por que "?":**

- Documentação da API não revisada
- Método de autenticação não claro
- Limites de taxa desconhecidos
- Acesso ao sandbox ausente

**Próximo Passo:**
Spike: Revisar documentação da API, testar sandbox.

---

## Exemplo 3: Questão de Viabilidade

**Título:** Otimização de Performance de Busca

**Descrição:**
A busca deve ser mais rápida.

**Por que "?":**

- Performance atual não medida
- Meta não definida (quão rápido?)
- Causa raiz do problema desconhecida

**Próximo Passo:**
Realizar análise de performance, coletar métricas.

---

## Abordagem Recomendada

1. **Adiar a história** – Não adicionar à sprint
2. **Planejar um spike** – Pesquisa com timebox (ex.: 4h, 1 dia)
3. **Documentar descobertas** – Registrar insights
4. **Re-estimar a história** – Estimar novamente com novo conhecimento
5. **Possivelmente dividir** – Dividir em histórias menores e estimáveis

---

## ⚠️ Nota

"?" **não é uma estimativa válida** para a sprint.
É um sinal para a equipe de que **trabalho prévio é necessário**.
