## 📋 示例 1：启用功能开关

### 用户故事

> 作为**产品负责人**，我希望**启用深色模式功能**，以便**用户可以立即开始使用**。

### 背景

深色模式功能已完全实现和测试，但在上次发布时被保留。现在需要通过更改功能开关来激活它。

### 实现

```json
// config/features.json
{
  "darkMode": true,  // ← 变更: false → true
  "betaFeatures": false,
  "newCheckout": true
}
```

### 验收标准

- [ ] 在 `config/features.json` 中将功能开关设置为 `true`
- [ ] 创建并合并 PR
- [ ] 触发部署
- [ ] 在生产环境测试深色模式

### 风险评估

| 方面     | 评级         |
| -------- | ------------ |
| 代码变更 | 1 行         |
| 测试     | 冒烟测试     |
| 回滚     | 重置功能开关 |

## 📋 示例 2：调整环境变量

### 用户故事

> 作为**运维工程师**，我希望**增加 API 超时时间**，以便**慢速 API 请求不再失败**。

### 背景

某些调用慢速第三方服务的 API 正在超时。当前 5 秒的超时应增加到 10 秒。

### 变更

```bash
# .env.production
API_TIMEOUT=10000  # 原值: 5000
```

### 验收标准

- [ ] 在 `.env.production` 中设置 `API_TIMEOUT=10000`
- [ ] 不需要代码变更
- [ ] 重新部署以激活
- [ ] 监控超时错误

---

## 📋 示例 3：修复拼写错误

### 用户故事

> 作为**用户**，我希望**在应用中看到正确的文字**，以便**产品看起来更专业**。

### 问题

网站页脚显示 **"Contcat"** 而不是 **"Contact"**。

### 解决方案

```vue
<!-- app/components/Footer.vue -->
<template>
  <footer>
    <a href="/contact">Contact</a>  <!-- 原为: Contcat -->
  </footer>
</template>
```

### 验收标准

- [ ] 修复 `Footer.vue` 中的拼写错误
- [ ] 检查其他地方是否有相同错误
- [ ] 在浏览器中进行视觉验证
