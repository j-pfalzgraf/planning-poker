# 0 ストーリーポイント – 些細な変更

> **工数:** 最小限、通常15分以内
> **リスク:** ほぼなし
> **テスト:** 通常不要
> **複雑さ:** なし

## 📋 例 1: 機能フラグの有効化

### ユーザーストーリー

> **プロダクトオーナー**として、**ダークモード機能を有効にしたい**。そうすることで、**ユーザーがすぐに利用を開始できます**。

### 背景

ダークモード機能は完全に実装・テスト済みですが、前回のリリースでは保留されていました。機能フラグを変更することで有効化する必要があります。

### 実装

```json
// config/features.json
{
  "darkMode": true,  // ← 変更: false → true
  "betaFeatures": false,
  "newCheckout": true
}
```

### 受け入れ基準

- [ ] `config/features.json` の機能フラグを `true` に設定する
- [ ] PRを作成してマージする
- [ ] デプロイをトリガーする
- [ ] 本番環境でダークモードをテストする

### リスク評価

| 観点         | 評価                 |
| ------------ | -------------------- |
| コード変更   | 1行                  |
| テスト       | スモークテスト       |
| ロールバック | 機能フラグをリセット |

## 📋 例 2: 環境変数の調整

### ユーザーストーリー

> **DevOpsエンジニア**として、**APIタイムアウトを延長したい**。そうすることで、**遅いAPIリクエストが失敗しなくなります**。

### 背景

遅いサードパーティサービスへの一部のAPI呼び出しがタイムアウトしています。現在の5秒のタイムアウトを10秒に延長する必要があります。

### 変更

```bash
# .env.production
API_TIMEOUT=10000  # 以前: 5000
```

### 受け入れ基準

- [ ] `.env.production` で `API_TIMEOUT=10000` を設定する
- [ ] コード変更は不要
- [ ] 再デプロイして有効化する
- [ ] タイムアウトエラーを監視する

## 📋 例 3: タイプミスの修正

### ユーザーストーリー

> **ユーザー**として、**アプリで正しいテキストを見たい**。そうすることで、**製品がプロフェッショナルに見えます**。

### 問題

ウェブサイトのフッターに **"Contcat"** と表示されていますが、正しくは **"Contact"** です。

### 解決策

```vue
<!-- app/components/Footer.vue -->
<template>
  <footer>
    <a href="/contact">Contact</a>  <!-- 以前: Contcat -->
  </footer>
</template>
```

### 受け入れ基準

- [ ] `Footer.vue` のタイプミスを修正する
- [ ] 他の箇所に同じエラーがないか確認する
- [ ] ブラウザで視覚的に確認する
