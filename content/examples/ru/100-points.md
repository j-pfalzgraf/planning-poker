# 100 Story Points – Монументальные изменения

> **Трудозатраты:** 1–3 месяца
> **Риск:** Экстремально высокий
> **Тесты:** Полная регрессия, тесты производительности и безопасности
> ⚠️ **ВНИМАНИЕ:** Должно быть разбито на эпики/истории!

---

## Пример 1: Полная миграция архитектуры

**Название:** Миграция с монолита на микросервисы

**Описание:**
Существующее монолитное приложение должно быть мигрировано на микросервисную архитектуру.

**Критерии приёмки:**

- [ ] Определены границы сервисов
- [ ] Реализован API Gateway
- [ ] Сервисы: Auth, User, Orders, Products, Notifications
- [ ] Событийная коммуникация (Kafka/RabbitMQ)
- [ ] Распределённая трассировка
- [ ] Оркестрация контейнеров (K8s)
- [ ] CI/CD для каждого сервиса
- [ ] Миграция данных
- [ ] Постепенная замена (Strangler Pattern)

---

## Пример 2: Мобильное приложение (кроссплатформенное)

**Название:** Нативное мобильное приложение для iOS и Android

**Описание:**
Разработка полноценного мобильного приложения со всеми основными функциями веб-приложения.

**Критерии приёмки:**

- [ ] Приложение React Native / Flutter
- [ ] Паритет функций с вебом (основные функции)
- [ ] Push-уведомления
- [ ] Оффлайн-режим
- [ ] Биометрическая аутентификация
- [ ] Публикация в App Store (iOS + Android)
- [ ] Deep linking
- [ ] Интеграция аналитики

---

## Пример 3: Корпоративный SSO и Compliance

**Название:** Пакет корпоративной безопасности

**Описание:**
Реализация функций корпоративной безопасности: SAML/OIDC SSO, SCIM provisioning, аудит-логи, инструменты соответствия GDPR.

**Критерии приёмки:**

- [ ] Поддержка SAML 2.0 и OIDC
- [ ] SCIM для provisioning пользователей
- [ ] Аудит-лог всех действий
- [ ] Экспорт данных (GDPR ст. 20)
- [ ] Удаление данных (GDPR ст. 17)
- [ ] Документация соответствия SOC 2
- [ ] Пройден тест на проникновение

---

## Почему 100 баллов?

- **Слишком большое для истории!**
- Должно быть разбито на эпики с множеством историй
- Месяцы разработки
- Трансформация архитектуры
- Наивысший риск
- Требуется выделенная команда

---

## ⚠️ Важное примечание

История на 100 баллов — это **не история** — это **проект** или **эпик**.

**Рекомендуемый подход:**

1. Разбить на эпики
2. Разбить эпики на истории (макс. 13 баллов)
3. Доставлять итеративно
4. Регулярно валидировать
