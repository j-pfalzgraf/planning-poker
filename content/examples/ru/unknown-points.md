# ? Story Points – Неизвестные трудозатраты

> **Трудозатраты:** Не могут быть оценены
> **Риск:** Неизвестен
> **Действие:** Требуется Spike/Исследование

---

## Когда используется "?"?

Карта "?" сигнализирует, что история **не может быть оценена**, потому что:

- Требования неясны
- Техническая осуществимость неопределённа
- Слишком много неизвестных
- Нужен дополнительный контекст

---

## Пример 1: Неясные требования

**Название:** Импорт данных из устаревшей системы

**Описание:**
Данные из старой системы должны быть импортированы.

**Почему "?":**

- В каком формате данные?
- Сколько записей?
- Какие поля маппятся?
- Есть ли правила валидации?

**Следующий шаг:**
Встреча со стейкхолдером для уточнения требований.

---

## Пример 2: Техническая неопределённость

**Название:** Интеграция со сторонним API

**Описание:**
Подключиться к API вендора XY.

**Почему "?":**

- Документация API не изучена
- Метод аутентификации неясен
- Rate limits неизвестны
- Sandbox-доступ отсутствует

**Следующий шаг:**
Spike: Изучить документацию API, протестировать sandbox.

---

## Пример 3: Вопрос осуществимости

**Название:** Оптимизация производительности поиска

**Описание:**
Поиск должен быть быстрее.

**Почему "?":**

- Текущая производительность не измерена
- Цель не определена (насколько быстрее?)
- Причина проблемы неизвестна

**Следующий шаг:**
Провести анализ производительности, собрать метрики.

---

## Рекомендуемый подход

1. **Отложить историю** – Не добавлять в спринт
2. **Запланировать spike** – Ограниченное по времени исследование (например, 4ч, 1 день)
3. **Задокументировать находки** – Записать выводы
4. **Переоценить историю** – Оценить снова с новыми знаниями
5. **Возможно разбить** – Разбить на меньшие, оцениваемые истории

---

## ⚠️ Примечание

"?" — это **не валидная оценка** для спринта.
Это сигнал команде, что **требуется предварительная работа**.
