# 3 Story Points – Малые и средние изменения

> **Трудозатраты:** 0,5–1 день
> **Риск:** Умеренный
> **Тесты:** Рекомендуются юнит- и интеграционные тесты

---

## Пример 1: Фильтруемый список

**Название:** Фильтрация списка задач по статусу

**Описание:**
Список задач должен фильтроваться по статусу (Открыто, В работе, Завершено). Кнопки фильтра над списком.

**Критерии приёмки:**

- [ ] Кнопки фильтра для каждого статуса
- [ ] Опция "Все"
- [ ] URL-параметр для фильтра (`?status=open`)
- [ ] Отображение пустого состояния при 0 результатах

---

## Пример 2: Форма с валидацией

**Название:** Форма регистрации с валидацией в реальном времени

**Описание:**
Поля: Email, Пароль, Подтверждение пароля. Валидация при потере фокуса и отправке.

**Критерии приёмки:**

- [ ] Валидация формата email
- [ ] Минимум 8 символов пароля
- [ ] Пароли должны совпадать
- [ ] Отображение ошибок inline

---

## Пример 3: Простая сортировка Drag & Drop

**Название:** Изменение порядка задач через Drag & Drop

**Описание:**
Задачи в списке должны быть переупорядочиваемы через drag & drop. Новый порядок сохраняется.

**Критерии приёмки:**

- [ ] Ручка перетаскивания на каждой задаче
- [ ] Визуальный предпросмотр при перетаскивании
- [ ] Сохранение после отпускания
- [ ] Поддержка сенсорных устройств

---

## Почему 3 балла?

- Несколько состояний для управления
- Изменения UI и логики
- Умеренная сложность
- Возможные граничные случаи
